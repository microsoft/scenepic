ScenePic: 3D Visualization Made Easy

© Microsoft 2021

Primary Contact: <matjoh@microsoft.com>

All platforms have good support for 2D images, with well-recognized formats such as PNG and JPEG that can be viewed out of the box (no installation) and shared trivially.

However, while many formats exist for 3D data, none are well-supported without installation of tools such as MeshLab, Blender, etc.

ScenePic was created for 3D computer vision researchers such as those working on [HoloLens](https://www.microsoft.com/en-gb/hololens) and [Mesh](https://www.microsoft.com/en-us/mesh) at Microsoft. It was designed to be a lightweight, reuseable 3D visualization library, with the following desiderata in mind:

  • Make experimentation with 3D data near effortless

  • Incredibly easy to create and share 3D results

    • zero-install sharing of detailed 3D results using HTML

    • based on modern web standards so usable with any modern browser

    • embeddable in other HTML documents

  • Performant

    • based on WebGL

  • High quality visuals

  • Works both offline or interactively in client-server setup

  • Simple, clean API

    • friendly Python front-end

    • basic mesh json file format

    • other language front ends easy to add

Getting Started

  1"""Example script demonstrating the basic ScenePic functionality."""
  2
  3import numpy as np
  4import scenepic as sp
  5
  6
  7def _main():
  8    # when we build a ScenePic we are essentially building a web
  9    # page, and the ScenePic will automatically populate parts of
 10    # that webpage.
 11
 12    # the scene object acts as the root of the entire ScenePic environment
 13    scene = sp.Scene()
 14
 15    # you can use it to create one or more canvases to display 3D or 2D
 16    # objects. Canvas objects display Frames. For a static ScenePic, there
 17    # will only be a single Frame, but you can use multiple frames to create
 18    # an animation or to display a range of visualizations in the same visual
 19    # space. We will create one 3D canvas to display the full scene, and then
 20    # some 2D canvases which will show projections of the scene.
 21    main = scene.create_canvas_3d(width=600, height=600)
 22    projx = scene.create_canvas_2d("projx", width=200, height=200)
 23    projy = scene.create_canvas_2d("projy", width=200, height=200)
 24    projz = scene.create_canvas_2d("projz", width=200, height=200)
 25
 26    # the scene object is also used to create Mesh objects that will be added
 27    # to frames. We are going to create an animation of some spheres orbiting
 28    # a fixed cube, so let's create a default unit cube to start.
 29    cube = scene.create_mesh("cube")
 30
 31    # the Mesh object has a variety of methods for adding primitive objects
 32    # or loading arbitrary mesh geometry. In this example, we will just
 33    # be using primitives, but the python tutorial shows all the various
 34    # methods for adding geometry to a mesh.
 35    cube.add_cube(color=sp.Colors.White)
 36
 37    # let's create our spheres as well, using some different colors
 38    sphere_names = ["red", "green", "blue"]
 39    sphere_colors = [sp.Colors.Red, sp.Colors.Green, sp.Colors.Blue]
 40    spheres = []
 41    for name, color in zip(sphere_names, sphere_colors):
 42        # by placing each sphere on a different layer, we can toggle them on and off
 43        sphere = scene.create_mesh("{}_sphere".format(name), layer_id=name)
 44        sphere.add_sphere(color=color, transform=sp.Transforms.scale(0.5))
 45        spheres.append(sphere)
 46
 47    # now we will iteratively create each frame of the animation.
 48    for i in range(180):
 49        # first we create a frame object. This will be used to populate
 50        # the 3D canvas.
 51        main_frame = main.create_frame()
 52
 53        # Now that we have a frame, we can add the cube mesh to the frame
 54        main_frame.add_mesh(cube)
 55
 56        # Next, we add the spheres. ScenePic has a variety of useful tools
 57        # for operating on 3D data. Some of the most useful enable us to
 58        # create transforms to move meshes around. Let's create the
 59        # transforms for our three rotating spheres and add them to the frame.
 60        # NB The Python interface uses numpy for matrices and vectors.
 61        positions = np.concatenate([np.eye(3, dtype=np.float32) * 1.3,
 62                                    np.ones((3, 1), dtype=np.float32)], axis=-1)
 63        inc = 2 * np.pi / 180
 64        positions[0] = sp.Transforms.RotationAboutYAxis(inc * i) @ positions[0].T
 65        positions[1] = sp.Transforms.RotationAboutZAxis(2 * inc * i) @ positions[1].T
 66        positions[2] = sp.Transforms.RotationAboutXAxis(3 * inc * i) @ positions[2].T
 67        positions = positions[:, :3]
 68        for sphere, position in zip(spheres, positions):
 69            transform = sp.Transforms.translate(position)
 70            main_frame.add_mesh(sphere, transform=transform)
 71
 72        # now we'll populate our projections
 73        for j, proj in enumerate([projx, projy, projz]):
 74            proj_frame = proj.create_frame()
 75
 76            # 2D frames work in pixels (as oppose to world units) so we need
 77            # to convert positions to pixels.
 78            proj_frame.add_rectangle(75, 75, 50, 50, fill_color=sp.Colors.White)
 79            points = np.roll(positions, j, axis=1)[:, 1:]
 80            points[:, 1] *= -1
 81            points = points * 50 + 100
 82
 83            for point, color in zip(points, sphere_colors):
 84                proj_frame.add_circle(point[0], point[1], 12.5, fill_color=color)
 85
 86            # let's add some label text
 87            proj_frame.add_text(proj.canvas_id, 10, 190, size_in_pixels=16)
 88
 89    # this will make user interactions happen to all canvases simultaneously
 90    scene.link_canvas_events(main, projx, projy, projz)
 91
 92    # ScenePic provides some useful layout controls by exposing CSS grid commands
 93    scene.grid(width="800px", grid_template_rows="200px 200px 200px", grid_template_columns="600px 200px")
 94    scene.place(main.canvas_id, "1 / span 3", "1")
 95    scene.place(projx.canvas_id, "1", "2")
 96    scene.place(projy.canvas_id, "2", "2")
 97    scene.place(projz.canvas_id, "3", "2")
 98
 99    # The scene is complete, so we write it to a standalone HTML file.
100    scene.save_as_html("getting_started.html", title="Getting Started")
101
102
103if __name__ == "__main__":
104    _main()

Using the HTML Client

A ScenePic HTML page will look something like the image above. This example shows four Canvas objects, each of which contains several Frame objects (refering under the hood to a set of Mesh objects).

The UI supports standard 3D mouse controls (drag to rotate, shift-drag to translate in xy, mousewheel to translate in z) to move the viewport camera. On touch screens: single finger to rotate, two fingers to translate in xyz. You can slow any of the mouse controls by holding down the Alt key. If you accidentally transform the camera too wildly, you can reset by pressing ‘r’.

In the top right of each canvas a Layer control will appear. This allows the user to toggle certain layers of meshes on and off interactively.

Each Frame for a 3D canvas has an associated Focus Point - a 3D location about which the camera rotates. You can view the Focus Point by holding down the ‘``’ (backtick) key, and while holding this down it can be translated using the mouse. If you press ‘l’ then the camera is ‘locked’ to the focus point in xy, though you can still rotate and translate in z. Locking to the focus point is particularly useful to center useful content in animations where the focus point can be different in each frame. You can also toggle the camera automatically orbiting the focus point by pressing ‘\’.

For Scene objects that contain many frames, you can control animation by pressing the spacebar to play/pause or by using the playback control widget. You can also use the scrollbar to select different frames.
